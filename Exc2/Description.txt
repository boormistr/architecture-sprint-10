1. Как данные хранятся
• ФИО, контакты, дата рождения:
	o Хранятся в зашифрованной базе данных (например, Postgres) с использованием AES-256.
	o Конфиденциальные данные разделены по доменам (PII, медицинские данные, финансовые данные), что минимизирует риск избыточного доступа.
• Медицинские карты и результаты анализов:
	o Хранятся в обезличенном виде, связаны с токенами, а не с реальными идентификаторами пациентов.
	o Оригинальные данные доступны только при строгой авторизации.
• Финансовые данные:
	o Хранятся в отдельной базе данных с дополнительным уровнем шифрования и изоляции.
	o Учет данных ведется в интеграции с «1С» через зашифрованные API.

2. Периоды и условия уничтожения данных
• Периоды уничтожения:
	o Данные пациентов хранятся в системе до 5 лет после завершения лечения в соответствии с законодательством РФ.
	o Лог-файлы доступа и операций удаляются через 1 год, если нет юридических причин их сохранения.
• Условия уничтожения:
	o По запросу клиента данные полностью удаляются из системы, включая резервные копии, в течение 30 дней.
	o Удаление осуществляется с помощью инструментов, поддерживающих безопасное стирание данных (например, алгоритмы DoD 5220.22-M).

3. Способы защиты данных при хранении и передаче
• При хранении:
	o Шифрование на уровне базы данных (Transparent Data Encryption).
	o Контейнеризация данных с использованием Role-Based Access Control (RBAC).
	o Маскирование данных в аналитических системах и отчётах.
• При передаче:
	o Используется TLS 1.3 для всех соединений.
	o Данные передаются через API, защищённые токенами OAuth 2.0.
	o Обезличивание данных перед передачей в лаборатории и партнёрские системы.

4. Усиление Data Privacy
• Тегирование данных:
	o Автоматическое добавление тегов PII, MEDICAL, FINANCIAL при создании записей в базе.
	o Использование этих тегов для ограничения доступа на уровне приложений.
• Мониторинг и аудит:
	o Внедрение системы SIEM для мониторинга доступа к данным.
	o Настройка алертов для выявления аномальной активности.
• Разграничение доступа:
	o Права доступа предоставляются строго в рамках роли (например, администратор, медицинский специалист, бухгалтер).
• Резервное копирование:
	o Резервные копии данных шифруются и хранятся отдельно от основной инфраструктуры.
	o Доступ к резервам возможен только после двухфакторной аутентификации.
